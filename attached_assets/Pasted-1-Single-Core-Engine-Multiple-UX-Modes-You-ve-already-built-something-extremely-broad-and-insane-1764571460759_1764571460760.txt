These are not different apps.

Same engine → multiple displays.

2. The switch is a single toggle

Just add:

user.interfaceMode:
- 'wellness'
- 'productivity'
- 'trader'
- 'team'
- 'therapy'


And you conditionally render components.

This prevents you from splitting your engineering effort.

7. The UX implementation approach

From the user’s perspective:

During onboarding, ask:

"How do you want to use JournOwl?"

Let them choose:

Self-reflection

Productivity

Emotional wellness

Performance optimization

Financial / trading psychology

Journaling for creativity

Team journaling

Then:
You don’t change the foundations.
You change:

UI language

Color theme

Homepage layout

Default prompts

Dashboard metrics

AI tone

Interaction style

Example:

Trader mode AI tone:
Direct, analytical, pattern-recognition

Poet / creative mode AI tone:
Empathetic, expressive, metaphorical

HR enterprise AI tone:
Neutral, professional, policy-compliant

This is just a system prompt difference.
